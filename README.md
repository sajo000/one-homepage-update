# 🚀 Deleo Front-end Project

이 프로젝트는 **Gulp** 빌드 시스템을 사용하여 리소스를 관리하며, `master` 브랜치 머지를 통해 실서버 배포가 이루어집니다.

---

## 🌿 브랜치 전략 및 배포 (Deployment)

* **브랜치 생성**: 수정 사항 발생 시 `master` 브랜치를 기준으로 **`front`** 브랜치를 생성하여 작업합니다.
* **실서버 배포**: `front` 브랜치에서 작업 완료 후 **`master`** 브랜치로 머지(Merge)하면 실서버에 자동으로 배포됩니다.

## 💻 개발 가이드 (Development)

### 1. 소스 수정
* 모든 수정 작업은 **`src/`** 폴더 내의 파일에서 진행합니다.

### 2. 결과물 확인
* 로컬 브라우저에서 작업 내용을 확인할 때는 **`dist/pages/`** 폴더 안에 있는 HTML 파일을 실행합니다.

## 🛠 실행 명령어 (Commands)

작업 시 터미널에서 아래 명령어를 실행하여 컴파일을 진행합니다.

```bash
# 1. 파일 빌드 및 컴파일 (src -> dist)
npm start

# 2. SCSS 컴파일
gulp
```


## ⚠️ 주의사항
dist/ 폴더 내의 파일은 컴파일된 결과물이므로 직접 수정하지 마세요. (빌드 시 수정 내용이 덮어씌워집니다.)

작업 시작 전 반드시 master 브랜치의 최신 상태를 유지하세요.
